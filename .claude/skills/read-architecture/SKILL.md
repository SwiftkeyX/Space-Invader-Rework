Reads all per-system GDDs and outputs a human-readable architecture summary: SRP per class and coupling to other classes with coupling type and strength. Writes the output to `docs/other/architecture-overview.md` (auto-generated — never edit manually).

---

## Agent

`claude`

---

## Docs

| Doc | Read/Write | Purpose |
|---|---|---|
| `.claude/docs/production/gdd/*.md` | Read | Extract SRP and coupling per system |
| `.claude/docs/other/architecture-overview.md` | Write | Auto-generated summary output |

---

## Steps

1. Read all GDD files in `.claude/docs/production/gdd/` (one per system)
2. For each GDD, extract:
   - **SRP**: the first sentence of the Summary section
   - **Coupling**: every row in the "Interactions with Other Systems" table — the partner system name, a short description of the interaction, and a coupling type + strength classification (see rules below)
3. Build the output (see format below)
4. Print the output to the user
5. Write the output to `.claude/docs/other/architecture-overview.md` with the AUTO-GENERATED banner

---

## Coupling Classification Rules

Classify each coupling entry by reading the Interaction description:

| Keyword in interaction | Type | Strength |
|---|---|---|
| "calls", "direct method call", "direct call" | Direct method call | **High** — caller must know callee's interface |
| "subscribes to", "event", "fires event", "static event" | C# event | **Low** — publisher and subscriber are decoupled |
| "Inspector", "serialized reference", "Inspector ref" | Inspector reference | **Medium** — wired by designer, not code; runtime tight |
| "reads", "property", "read-only" | Property read | **Low** — passive read, no mutation |

If a single interaction involves BOTH a direct call AND an event subscription (e.g. "calls X and subscribes to Y"), list both types and use the highest strength.

---

## Output Format

```
> AUTO-GENERATED by /read-architecture — do not edit manually. Re-run to refresh.
> Last generated: <date>

---

## <SystemName>
**SRP**: <first sentence of Summary>

| Coupled To | Type | Strength | Reason |
|---|---|---|---|
| <SystemName> | Direct method call | High | Calls `Method(args)` |
| <SystemName> | C# event | Low | Subscribes to `OnEventName` |
| <SystemName> | Inspector reference | Medium | Reads `Property` via serialized ref |
```

List systems with NO coupling last, noted as "(none — standalone)".

---

## Constraints

- Never edit any GDD file
- Never treat `architecture-overview.md` as a source of truth — it is always derived from GDDs
- If a GDD is missing, add a row `| <SystemName> | GDD NOT FOUND | — | — |` and continue
- Keep the Reason column short (one method name or event name is enough — not the full interaction description)
